# 安全策略

## 支持范围

本项目只维护 `main` 分支的最新版本。历史提交、个人分叉和独立展示项目不在本仓库的安全支持范围内。

## 报告漏洞

请不要通过公开 Issue、讨论区或日志粘贴漏洞细节、凭据、Cookie、备份内容或个人数据。

优先使用 GitHub 仓库 `Security` 页面中的私密漏洞报告功能。若该入口不可用，请先通过维护者 GitHub 主页中的私密联系方式建立联系，仅在确认安全通道后发送复现细节。

报告应尽量包含：

- 受影响的提交、版本或线上路径。
- 最小复现步骤与实际影响。
- 是否已经访问、修改或删除真实数据。
- 建议的修复方向，以及是否需要协调披露时间。

维护者确认收到后会先评估影响和临时缓解措施，再协调修复与披露。不得为验证漏洞而扩大访问范围、持续探测、读取无关数据或影响线上可用性。

## 凭据泄露

疑似泄露的数据库密码、管理员凭据、会话签名密钥和外部 AI Key 必须立即在对应系统轮换。仅从 Git 删除文件不能使已经暴露的凭据失效。

安全报告和自动化日志不得包含 `.env` 内容、完整 Cookie、Authorization 字段、secret 文件内容或数据库转储。
